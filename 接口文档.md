# 个人发展规划网站 — 接口文档

> 版本：v1.0 | 日期：2026-04-23 | 协议：RESTful + JSON
>
> 统一响应格式：`{ code: number, msg: string, data: any }`，code=1 成功，code=0 失败
>
> 认证方式：JWT，登录成功后获取 token，后续请求在 Header 中携带 `token`

## 1. 认证模块(已完成)

### 1.1 用户注册

#### 1.1.1 基本信息

> 请求路径：/auth/register
>
> 请求方式：POST
>
> 接口描述：用户注册账号

#### 1.1.2 请求参数

格式：application/json

参数说明：

| 参数名   | 类型   | 是否必须 | 备注   |
| -------- | ------ | -------- | ------ |
| username | string | 必须     | 用户名 |
| email    | string | 必须     | 邮箱   |
| password | string | 必须     | 密码   |

请求数据样例：

```json
{
  "username": "xiaoming",
  "email": "xiaoming@example.com",
  "password": "123456"
}
```

#### 1.1.3 响应数据

参数格式：application/json

参数说明：

| 参数名   | 类型   | 是否必须 | 备注     |
| -------- | ------ | -------- | -------- |
| code     | number | 必须     | 1成功 0失败 |
| msg      | string | 非必须   | 提示信息 |
| data     | object | 非必须   | 返回的数据 |
| \|- id   | number | 非必须   | 用户ID   |
| \|- username | string | 非必须   | 用户名   |
| \|- email    | string | 非必须   | 邮箱     |
| \|- token    | string | 非必须   | JWT令牌  |

响应数据样例：

```json
{
  "code": 1,
  "msg": "注册成功",
  "data": {
    "id": 1,
    "username": "xiaoming",
    "email": "xiaoming@example.com",
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

### 1.2 用户登录

#### 1.2.1 基本信息

> 请求路径：/auth/login
>
> 请求方式：POST
>
> 接口描述：用户登录，登录成功后下发 JWT 令牌

#### 1.2.2 请求参数

格式：application/json

参数说明：

| 参数名   | 类型   | 是否必须 | 备注              |
| -------- | ------ | -------- | ----------------- |
| username | string | 必须     | 用户名或邮箱      |
| password | string | 必须     | 密码              |

请求数据样例：

```json
{
  "username": "xiaoming",
  "password": "123456"
}
```

#### 1.2.3 响应数据

参数格式：application/json

参数说明：

| 参数名       | 类型   | 是否必须 | 备注         |
| ------------ | ------ | -------- | ------------ |
| code         | number | 必须     | 1成功 0失败  |
| msg          | string | 非必须   | 提示信息     |
| data         | object | 非必须   | 返回的数据   |
| \|- id       | number | 非必须   | 用户ID       |
| \|- username | string | 非必须   | 用户名       |
| \|- realName | string | 非必须   | 真实姓名     |
| \|- avatar   | string | 非必须   | 头像URL      |
| \|- token    | string | 非必须   | JWT令牌      |

响应数据样例：

```json
{
  "code": 1,
  "msg": "登录成功",
  "data": {
    "id": 1,
    "username": "xiaoming",
    "realName": "小明",
    "avatar": "https://example.com/avatar.jpg",
    "token": "eyJhbGciOiJIUzI1NiJ9..."
  }
}
```

#### 1.2.4 备注说明

> 用户登录成功后，系统会自动下发 JWT 令牌，然后在后续的每次请求中，都需要在请求头 header 中携带到服务端，请求头的名称为 `token`，值为登录时下发的 JWT 令牌。
>
> 如果检测到用户未登录，则直接响应 401 状态码。

### 11.3 导入导出记录查询

#### 11.3.1 基本信息

> 请求路径：/data/operations
>
> 请求方式：GET
>
> 接口描述：查询数据导入导出操作历史记录，支持条件筛选

#### 11.3.2 请求参数

参数格式：queryString

参数说明：

| 参数名称      | 是否必须 | 示例   | 备注                                       |
| ------------- | -------- | ------ | ------------------------------------------ |
| operationType | 否       | import | 操作类型：import/export                    |
| status        | 否       | success | 状态：pending/success/fail                |
| page          | 是       | 1      | 分页查询的页码，如果未指定，默认为1        |
| pageSize      | 是       | 10     | 分页查询的每页记录数，如果未指定，默认为10 |

请求数据样例：

```shell
/data/operations?operationType=import&status=success&page=1&pageSize=10
```

#### 11.3.3 响应数据

参数格式：application/json

参数说明：

| 名称              | 类型      | 是否必须 | 备注                           |
| ----------------- | --------- | -------- | ------------------------------ |
| code              | number    | 必须     | 1成功 0失败                    |
| msg               | string    | 非必须   | 提示信息                       |
| data              | object    | 必须     | 返回的数据                     |
| \|- total         | number    | 必须     | 总记录数                       |
| \|- rows          | object [] | 必须     | 数据列表                       |
| \|- id            | number    | 非必须   | 记录ID                         |
| \|- operationType | string    | 非必须   | 操作类型：import/export        |
| \|- dataType      | string    | 非必须   | 数据类型                       |
| \|- fileUrl       | string    | 非必须   | 文件地址                       |
| \|- status        | string    | 非必须   | 状态                           |
| \|- createdAt     | string    | 非必须   | 创建时间                       |

响应数据样例：

```json
{
  "code": 1,
  "msg": "success",
  "data": {
    "total": 2,
    "rows": [
      {
        "id": 1,
        "operationType": "import",
        "dataType": "课程",
        "fileUrl": "/uploads/import/courses.xlsx",
        "status": "success",
        "createdAt": "2026-04-20T10:00:00"
```
