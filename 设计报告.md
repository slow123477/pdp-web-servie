### 1.3 主要用例说明

#### UC-01：用户注册

| 项目 | 说明 |
|------|------|
| 参与者 | 访客 |
| 前置条件 | 还没注册过 |
| 基本流程 | 1. 填写用户名、邮箱、密码<br>2. 后端检查用户名和邮箱是否已经被注册<br>3. 密码用 BCrypt 加密后存到数据库<br>4. 注册成功，返回 JWT Token |
| 异常情况 | 用户名已存在 → 提示"用户名已存在"<br>邮箱已存在 → 提示"邮箱已被注册" |

#### UC-02：用户登录

| 项目 | 说明 |
|------|------|
| 参与者 | 访客、学生用户 |
| 基本流程 | 1. 输入用户名/邮箱和密码<br>2. 后端校验密码<br>3. 生成 JWT Token 返回<br>4. 前端存下 Token，跳转到仪表盘 |
| 异常情况 | 账号或密码错误 → 提示"用户名或密码错误"<br>Token 过期 → 401，前端自动跳转登录页 |

#### UC-03：AI 智能分析（这是我们项目的特色功能）

| 项目 | 说明 |
|------|------|
| 参与者 | 学生用户 |
| 前置条件 | 至少录入了一条课程或经历数据 |
| 基本流程 | 1. 选择想分析的维度（能力评估、发展方向等）<br>2. 后端把用户的课程、经历、成就、角色数据整理成 Prompt<br>3. 调用 Kimi AI API 进行分析<br>4. 解析 AI 返回的 JSON，保存到数据库<br>5. 前端展示雷达图和报告 |
| 异常情况 | AI 服务超时 → 提示"AI 服务暂时不可用，请稍后重试"<br>没数据 → 提示"快去录入你的课程、经历和成就数据吧" |

## 3. 详细设计

### 3.1 后端架构

后端采用 Spring Boot + Maven 多模块结构，分成三层：

```
┌──────────────────────────────────────┐
│         Controller 层                 │
│   接收 HTTP 请求，返回 Result<T>      │
│   用 @Tag / @Operation 写接口文档    │
├──────────────────────────────────────┤
│          Service 层                   │
│   处理业务逻辑，DTO 和 Entity 转换   │
├──────────────────────────────────────┤
│          Mapper 层                    │
│   MyBatis 访问数据库                 │
├──────────────────────────────────────┤
│         MySQL 数据库                  │
│   数据库名：personal_dev_plan        │
└──────────────────────────────────────┘
```

### 3.2 核心类说明

#### Result 统一响应类

所有接口都返回这个结构，方便前端统一处理：

```java
public class Result<T> {
    private Integer code;   // 1=成功, 0=失败
    private String msg;
    private T data;
}
```

#### JWT 工具类

登录成功后生成 Token，后续请求都要带上：

```java
public class JwtUtil {
    // 生成令牌，默认有效期 12 小时
    public static String generateToken(Map<String, Object> claims) { ... }
    // 解析令牌
    public static Claims parseToken(String token) { ... }
}
```

Token 放在请求头的 `token` 字段里传给后端。

#### 全局异常处理

用 `@RestControllerAdvice` 统一捕获异常，避免每个接口都写 try-catch：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        return Result.error(e.getMessage());
    }
}
```

### 3.3 模块接口一览

| 模块 | 接口数量 | 主要功能 |
|------|---------|---------|
| 认证 | 3 | 注册、登录、获取当前用户 |
| 用户 | 4 | 查用户、改资料、改密码、上传头像 |
| 仪表盘 | 1 | 聚合统计所有核心数据 |
| 课程 | 5 | 增删改查 + 分页 |
| GPA | 2 | 计算 GPA、趋势数据 |
| 经历 | 5 | 增删改查 + 附件上传 |
| 成就 | 5 | 增删改查 + 证书上传 |
| 角色 | 5 | 增删改查 |
| AI分析 | 4 | 生成报告、历史列表、详情、删除 |
| 设置 | 2 | 查设置、改设置 |
| 数据 | 3 | 导入、导出、操作记录 |
| 文件 | 1 | 通用上传 |

### 3.4 GPA 计算逻辑

支持三种计算标准，用户可以在设置页切换：

- **4.0 制**：90-100 分 = 4.0，85-89 = 3.7，以此类推
- **5.0 制**：90-100 分 = 5.0，85-89 = 4.5，以此类推
- **百分制加权**：直接按 `Σ(成绩×学分)/Σ(学分)` 计算

绩点对照表存在 `user_settings` 表的 `gpa_scale` JSON 字段里，用户可以自定义修改。

## 6. 遇到的问题和解决办法

### 6.1 登录后页面不跳转

**现象**：输入账号密码提示登录成功，但页面一直停在登录页。

**原因**：`router.push` 执行时 Pinia 的 token 还没同步完成，路由守卫拦截了跳转。

**解决**：在 `login/index.vue` 里用 `setTimeout` 延迟 50ms 再执行 `router.push`，同时加了 `window.location.href` 兜底方案。

### 6.2 GPA 柱状图高度溢出

**现象**：选择"百分制加权"后，仪表盘的成绩柱状图高度变成了几千像素，把页面撑坏了。

**原因**：柱状图高度按 `gpa / 4.0 * 100%` 计算，百分制 GPA 可能到 95，导致高度 2375%。

**解决**：改为按数据集中的最大 GPA 做归一化，`(gpa / maxGpa * 100%)`。

### 6.3 AI 摘要显示 HTML 标签

**现象**：AI 返回的报告内容里有 `<strong>`、`<p>` 等标签，直接显示在页面上。

**原因**：Vue 的 `{{ }}` 插值会自动转义 HTML，应该用 `v-html` 渲染。

**解决**：改成 `v-html`，同时加了 `sanitizeHtml` 函数过滤掉 `<script>` 等危险标签防止 XSS。

### 6.4 AI 服务调用超时

**现象**：点击生成分析报告后长时间没反应，最后报错。

**原因**：AI 接口响应慢，默认 10 秒超时不够。

**解决**：AI 分析请求单独设置 `timeout: 120000`（120秒），后端也加了 try-catch，失败时返回友好提示。

## 7. 安全设计

### 7.1 认证与授权

- 登录后下发 JWT Token，有效期 12 小时
- 前端每次请求在 Header 里带上 `token`
- Token 无效或过期返回 401，前端自动跳转登录页
- 密码用 BCrypt 加密存储

### 7.2 输入安全

- AI 返回的 HTML 内容在前端渲染前会过滤 `<script>`、`<iframe>`、`javascript:` 等
- 后端 DTO 用 `@Valid` 做参数校验
- MyBatis 用 `#{}` 预编译，防止 SQL 注入

### 7.3 文件上传

- 限制文件类型：图片只能是 jpg/png，导入只能是 csv/xlsx/xls
- 单个文件最大 10MB
- 上传文件用 UUID 重命名，防止文件名冲突
