# 个人发展规划网站 — 详细设计分工报告

> 版本：v1.0 | 日期：2026-04-24 | 团队成员：5人

---

## 一、项目概述

本项目为教育类个人成长管理平台，面向在校大学生，采用 **前后端分离架构** 开发。

- **前端技术栈**：Vue 3 + Vite + Element Plus + Axios + Pinia
- **后端技术栈**：Spring Boot + Maven + MyBatis + PageHelper + JWT
- **数据库**：MySQL 8.0+
- **AI 服务**：Spring AI + Kimi AI API（Moonshot）

---

## 二、分工原则

1. **前后端同责**：每个成员负责所分配模块的完整前端页面 + 后端接口，减少接口联调成本
2. **模块解耦**：各模块之间依赖关系低，支持并行独立开发
3. **工作量均衡**：综合页面复杂度、后端逻辑难度、技术挑战度进行分配
4. **基础先行**：指定一名成员负责项目初始化、公共基础设施搭建及数据库创建

## 五、协作约定

| 事项 | 具体约定 |
|------|----------|
| **接口规范** | 严格按照《接口文档.md》中定义的请求路径、请求方式、参数格式、响应格式开发，code=1 表示成功，code=0 表示失败 |
| **数据库** | 统一使用成员1初始化后的数据库 `personal_dev_plan`，表结构以《数据库设计.md》为准，开发过程中不随意修改表结构 |
| **前端路由** | 统一采用《需求文档》中的路径规范：`/dashboard`、`/courses`、`/experiences`、`/achievements`、`/roles`、`/ai-analysis`、`/settings`、`/data` |
| **状态管理** | 用户登录态（token、用户信息）使用 Pinia 全局存储，前端路由通过导航守卫拦截未登录请求 |
| **JWT 传递** | 前端所有需认证的请求必须在 Header 中携带 `token: <JWT令牌>`，后端统一从请求头解析并校验 |
| **Git 协作** | 主分支为 `main`，每人从 `main` 检出 `feature/模块名` 分支进行开发，完成后提交 Pull Request 合并 |
| **开发顺序** | 第1天：成员1完成项目初始化；其余成员并行熟悉文档和原型。第2天起：所有成员并行开发各自模块 |
| **联调测试** | 各模块开发完成后，统一在成员1搭建的环境上进行前后端联调，确保接口数据流通正常 |
