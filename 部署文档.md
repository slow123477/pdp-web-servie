# 个人发展规划网站 — 部署文档

> 本文档指导您在 Windows 本地环境中完成项目的环境搭建与启动运行。

## 2. 项目结构说明

```
个人发展规划网站/
├── pdp-web/                    # 前端项目（Vue 3 + Vite）
│   ├── src/                    # 前端源码
│   ├── package.json            # 前端依赖配置
│   └── vite.config.js          # Vite 配置文件
├── pdp-service/                # 后端项目（Spring Boot + Maven 多模块）
│   ├── common/                 # 公共工具模块
│   ├── pojo/                   # 实体/DTO/VO 模块
│   ├── service/                # 主应用模块
│   │   ├── src/main/resources/
│   │   │   ├── application.yml # 后端核心配置文件
│   │   │   └── mapper/         # MyBatis XML 映射文件
│   │   └── pom.xml
│   └── pom.xml                 # 父 POM
├── sql/
│   └── sqlfile.sql             # 数据库初始化脚本
├── 接口文档.md
├── 数据库设计.md
└── 需求文档-个人发展规划网站-2026-04-22.md
```

**架构说明**：本项目采用前后端分离架构。
- 前端运行在 **5173** 端口，通过代理将 `/api` 请求转发到后端。
- 后端运行在 **8080** 端口，提供 RESTful API、文件上传和 Swagger 接口文档。

## 3. 数据库部署

### 3.1 启动 MySQL 服务

1. 打开 **MySQL Workbench** 或 **CMD**。
2. 确保 MySQL 服务已启动（可在"服务"管理器中查看 `MySQL80` 状态）。

### 3.2 执行初始化脚本

**方式一：通过 MySQL Workbench（推荐）**

1. 打开 MySQL Workbench，连接到本地 MySQL 实例（默认端口 3306）。
2. 点击菜单 `File → Open SQL Script`，选择项目目录下的 `sql/sqlfile.sql`。
3. 点击执行按钮（闪电图标）。

**方式二：通过命令行**

```bash
# 进入项目根目录，执行以下命令
mysql -u root -p < sql/sqlfile.sql
```

输入 MySQL root 密码后执行。脚本将自动完成以下操作：
- 创建数据库 `personal_dev_plan`
- 创建 8 张业务表（users、courses、experiences、achievements、roles、ai_analysis、user_settings、data_operations）
- 设置字符集为 `utf8mb4`（支持中文和 Emoji）

### 3.3 验证数据库

```sql
SHOW DATABASES;
USE personal_dev_plan;
SHOW TABLES;
```

应能看到 8 张表，表示数据库初始化成功。

## 4. 后端服务部署

### 4.1 配置修改

打开文件 `pdp-service/service/src/main/resources/application.yml`，检查并修改以下配置项：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/personal_dev_plan?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root          # 如果您的 MySQL 用户名不是 root，请修改
    password: 123456        # 修改为您的 MySQL root 密码
```

> **注意**：如果 MySQL 端口不是 3306，或数据库不在本机，请同步修改 `url` 中的地址和端口。

### 4.2 Maven 编译

打开 CMD 或 PowerShell，进入后端项目根目录：

```bash
cd pdp-service
mvn clean compile
```

**预期输出**：
```
[INFO] BUILD SUCCESS
[INFO] Total time:  30.123 s
```

> **首次编译**会自动从 Maven 中央仓库下载依赖，耗时约 3~10 分钟，请耐心等待。

### 4.3 启动服务

继续在 `pdp-service` 目录下执行：

```bash
mvn spring-boot:run -pl service
```

**启动成功标志**：控制台输出包含以下信息：
```
Started ServiceApplication in X.XXX seconds
Tomcat started on port(s): 8080 (http)
```

> **保持此窗口运行**，关闭窗口即停止服务。

### 4.4 验证后端

打开浏览器访问：http://localhost:8080/doc.html

应能看到 Knife4j 接口文档页面，表示后端启动成功。

## 5. 前端部署

### 5.1 安装依赖

**新开一个 CMD / PowerShell 窗口**（不要关闭后端窗口），进入前端目录：

```bash
cd pdp-web
npm install
```

> **首次安装**会自动从 npm 仓库下载依赖，耗时约 2~5 分钟。

### 5.2 开发模式启动

```bash
npm run dev
```

**启动成功标志**：
```
VITE v6.x.x  ready in XXX ms

  ➜  Local:   http://localhost:5173/
```

### 5.3 访问系统

打开浏览器，访问：http://localhost:5173

首次访问将自动跳转至登录页。您可以：
- 点击"立即注册"创建新账号
- 使用注册账号登录后使用全部功能

> **保持此窗口运行**，关闭窗口即停止前端服务。

### 5.4 生产构建（可选）

如需构建生产环境静态文件：

```bash
npm run build
```

构建产物将输出到 `pdp-web/dist/` 目录。

## 6. AI 功能配置

AI 智能分析模块需要调用大模型 API。未配置时，AI 分析功能将不可用，**其他所有功能不受影响**。

### 6.1 获取阿里云百炼 API Key

本项目使用阿里云百炼平台（兼容 OpenAI 协议）提供的 Kimi 大模型服务。

**步骤如下**：

1. **注册/登录阿里云账号**
   - 访问 https://www.aliyun.com/
   - 使用手机号或支付宝扫码注册并登录

2. **开通百炼大模型服务**
   - 访问 https://bailian.console.aliyun.com/
   - 首次使用需同意服务协议并开通服务（免费额度内无需付费）

3. **创建 API Key**
   - 在百炼控制台左侧菜单选择「API Key 管理」
   - 点击「创建新的 API Key」
   - 复制生成的 Key（格式如 `sk-xxxxxxxxxxxxxxxx`）
   - ![image-20260429170351802](E:\Software\Typora\images\image-20260429170351802.png)
   
4. **确认模型可用**
   - 在控制台「模型广场」确认 `kimi-k2.6` 模型已开通（通常默认开通）
   - ![image-20260429170500973](E:\Software\Typora\images\image-20260429170500973.png)

> **费用说明**：阿里云百炼提供一定的免费调用额度，超出后按量计费。交付代码后，后续使用需客户自行承担 API 费用。

### 6.2 配置 API Key

获取 API Key 后，有两种配置方式：

**方式一：环境变量（推荐）**

在 Windows 系统中设置环境变量：

```powershell
# PowerShell 中执行（临时生效，仅当前窗口）
$env:ALIYUN_API_KEY="sk-xxxxxxxxxxxxxxxx"

# 然后重新启动后端服务
```

或在「系统属性 → 高级 → 环境变量」中新建系统变量：
- 变量名：`ALIYUN_API_KEY`
- 变量值：`sk-xxxxxxxxxxxxxxxx`

设置后需**重启 CMD/PowerShell 窗口**，再重新启动后端服务。

**方式二：直接修改配置文件**

编辑 `pdp-service/service/src/main/resources/application.yml`：

```yaml
spring:
  ai:
    openai:
      api-key: sk-xxxxxxxxxxxxxxxx    # 将 ${ALIYUN_API_KEY} 替换为实际 Key
      base-url: https://dashscope.aliyuncs.com/compatible-mode
```

> **注意**：直接修改配置文件可能导致 Key 意外提交到代码仓库，建议使用环境变量方式。

### 6.3 验证 AI 功能

配置完成后，重启后端服务，登录系统后进入「AI 智能分析」页面：
- 选择分析维度
- 点击「生成分析报告」
- 等待约 10~30 秒，应能看到 AI 生成的分析报告

## 8. 常见问题排查

### 8.1 端口被占用

**现象**：启动后端或前端时报错 `Port 8080/5173 was already in use`

**解决**：
```powershell
# 查找占用 8080 端口的进程
netstat -ano | findstr :8080

# 结束对应进程（PID 为最后一列数字）
taskkill /PID <PID> /F
```

或修改后端端口（`application.yml` 中 `server.port`）或前端端口（`vite.config.js` 中 `server.port`）。

### 8.2 数据库连接失败

**现象**：后端启动报错 `Communications link failure` 或 `Access denied`

**排查步骤**：
1. 确认 MySQL 服务已启动（`services.msc` 中查看 `MySQL80`）
2. 确认 `application.yml` 中的数据库地址、用户名、密码正确
3. 确认数据库 `personal_dev_plan` 已创建（`SHOW DATABASES;`）

### 8.3 前端无法访问后端接口

**现象**：前端页面报错 `Network Error` 或接口返回 404/500

**排查步骤**：
1. 确认后端服务已启动且运行在 8080 端口
2. 确认前端 `vite.config.js` 中代理配置正确：
   ```js
   proxy: {
     '/api': {
       target: 'http://localhost:8080',
       changeOrigin: true,
     }
   }
   ```
3. 检查浏览器开发者工具（F12 → Network）查看具体请求报错

### 8.4 AI 分析功能报错

**现象**：点击"生成报告"后提示"AI 服务异常"或长时间无响应

**排查步骤**：
1. 确认已正确配置 `ALIYUN_API_KEY` 环境变量
2. 确认环境变量生效（重启后端启动窗口）
3. 检查后端控制台日志，确认是否显示 `401 Unauthorized`（Key 无效）或连接超时
4. 确认网络可正常访问 `https://dashscope.aliyuncs.com`

### 8.5 文件上传失败

**现象**：上传头像或附件时报错

**排查步骤**：
1. 确认上传文件不超过 10MB（`application.yml` 中 `max-file-size` 限制）
2. 确认后端启动用户对上传目录有写入权限
3. 上传目录默认为 `${user.dir}/service/src/main/resources/static/upload`

### 8.6 Maven 下载依赖慢

**现象**：`mvn clean compile` 长时间卡在下载阶段

**解决**：在 `pdp-service/pom.xml` 同级目录创建 `settings.xml` 或修改用户目录下的 `.m2/settings.xml`，配置阿里云 Maven 镜像：

```xml
<settings>
  <mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>
```

## 9. 服务停止

- **停止后端**：在运行后端的 CMD/PowerShell 窗口中按 `Ctrl + C`，然后输入 `Y` 确认。
- **停止前端**：在运行前端的 CMD/PowerShell 窗口中按 `Ctrl + C`，然后输入 `Y` 确认。
- **停止 MySQL**：可在「服务」管理器中停止 `MySQL80` 服务，或保持运行。
