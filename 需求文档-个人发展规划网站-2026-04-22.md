# 个人发展规划网站（Personal Development Planning Website）- 需求文档

> 版本：v1.0 | 日期：2026-04-22

---

## 1. 项目概述

### 1.1 项目背景

本项目为 **教育类个人成长管理平台**，面向在校大学生，旨在帮助学生记录和追踪四年学习历程。项目围绕大学生的个人成长与发展需求展开，帮助用户对自己的学习、能力、经历与未来目标进行记录、分析、规划与展示。

### 1.2 项目范围

- **包含**：本文档所列全部功能模块、页面、技术实现及交付物
- **不包含**：超出本文档范围的需求变更、第三方服务费用（如服务器、域名、短信、支付手续费等）

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 课程记录模块
- **功能描述**：学生可添加、编辑、删除所学课程信息
- **输入**：课程名称、学分、学期、成绩、课程类型等
- **处理**：数据校验后存入数据库，支持按学期/学年筛选
- **输出**：课程列表、学期汇总统计
- **异常处理**：重复课程名称提示、必填项校验

#### 2.1.2 竞赛 / 项目 / 实习经历模块
- **功能描述**：记录学生参加的竞赛、项目、实习经历
- **输入**：经历名称、类型（竞赛/项目/实习）、起止时间、描述、成果、证明材料等
- **处理**：支持时间线展示，可上传相关图片或文件
- **输出**：经历列表、时间线视图
- **异常处理**：文件格式/大小限制、时间逻辑校验

#### 2.1.3 成绩与 GPA 计算模块
- **功能描述**：录入课程成绩，自动计算 GPA
- **输入**：课程成绩、学分、GPA 计算方式（4.0 制 / 5.0 制 / 百分制加权等）
- **处理**：根据所选算法自动计算 GPA，生成趋势图表
- **输出**：GPA 报表、学期 GPA、累计 GPA、成绩趋势图
- **异常处理**：成绩范围校验、计算方式配置校验

#### 2.1.4 成就记录模块
- **功能描述**：记录获得的奖项、奖金、荣誉等
- **输入**：成就名称、级别（校级/省级/国家级等）、时间、颁发机构、描述、证明图片
- **处理**：支持分类标签（学术/竞赛/志愿等），可上传证书图片
- **输出**：成就列表、分类统计
- **异常处理**：文件格式/大小限制

#### 2.1.5 担任角色模块
- **功能描述**：记录学生担任的各类角色
- **输入**：角色名称（班级代表、模块代表、志愿者、社团职务等）、起止时间、职责描述
- **处理**：按时间线展示，计算任职时长
- **输出**：角色列表、时间线视图
- **异常处理**：时间逻辑校验

#### 2.1.6 AI 智能分析模块（创新）
- **功能描述**：将学生的课程、经历、成就、角色等数据发送给 AI 进行分析，生成个人发展分析报告
- **输入**：学生的综合经历数据（课程记录、竞赛/项目/实习经历、成就、担任角色等），可选择分析维度（能力评估、发展方向、简历优化建议等）
- **处理**：后端通过 Spring AI 调用 Kimi AI API，将结构化数据组织为 Prompt 发送，接收并解析 AI 返回的分析结果
- **输出**：AI 生成的个人发展分析报告（能力雷达图分析、优势与不足、发展建议、简历亮点提炼等），支持保存历史分析记录
- **异常处理**：AI 服务异常时返回友好提示、数据为空时提示完善信息、分析超时处理

### 2.2 隐含需求模块

#### 2.2.1 用户注册 / 登录
- **功能描述**：用户账号的注册、登录、退出
- **技术实现**：JWT 身份认证，密码加密存储

#### 2.2.2 个人资料管理
- **功能描述**：修改个人信息、头像、密码等

#### 2.2.3 数据导入 / 导出
- **功能描述**：支持 CSV / Excel 格式的数据导入导出
- **适用范围**：课程、成绩、经历等模块的数据批量操作

#### 2.2.4 仪表盘概览
- **功能描述**：统计总学分、总成就数、GPA 趋势、近期动态等核心数据的可视化展示

### 2.3 用户角色与权限

| 角色 | 权限说明 |
|------|---------|
| 普通用户（学生） | 管理个人课程、成绩、经历、成就、角色等全部个人数据 |
| 访客 | 仅可访问登录/注册页面，需登录后使用功能 |

## 4. 技术方案

### 4.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端框架 | Vue 3 + Vite |
| UI 组件库 | Element Plus |
| 后端框架 | Spring Boot + Maven |
| ORM / 数据访问 | MyBatis + PageHelper |
| 数据库 | MySQL |
| AI 框架 | Spring AI |
| AI 服务 | Kimi AI API（Moonshot）可更改 |
| 接口文档 | Swagger |
| 身份认证 | JWT |
| 部署方式 | 本地运行 |

### 4.2 系统架构

本项目采用前后端分离架构：
- **前端**：Vue 3 单页应用，通过 Axios 与后端 API 交互，使用 Element Plus 组件库构建界面
- **后端**：Spring Boot 提供 RESTful API，MyBatis 负责数据持久化，PageHelper 实现分页查询
- **AI 模块**：Spring AI 封装 Kimi AI API 调用，将学生经历数据构建为 Prompt 发送给 AI，解析返回的分析报告
- **数据库**：MySQL 存储业务数据，AI 分析历史记录持久化存储
- **认证机制**：基于 JWT 的无状态身份认证

### 4.3 第三方服务集成

| 服务类型 | 提供商 | 用途 | 费用承担方 |
|---------|--------|------|-----------|
| AI 大模型服务 | Moonshot（Kimi AI） | 学生经历智能分析、生成发展报告 | 开发方 |

> 注：AI 服务为功能演示使用，开发中产生的 API 费用由开发方承担。交付代码后客户后续使用需自行准备API密钥，开发方后续将不再提供API。

### 5.1 交付物清单

- [x] 完整源代码（前后端）

- [x] 数据库设计文档
- [x] 详细设计分工报告
- [x] 项目时间规划 / 甘特图

### 5.2 验收标准

1. 所有功能模块按本文档实现并通过功能测试
2. 可执行文件无需修改代码即可直接运行展示
3. 代码提交记录完整，分工明确
4. 接口文档通过 Swagger 可正常访问
5. 主要浏览器兼容性：Chrome 90+ / Safari 14+
6. 页面首屏加载时间 <= 3 秒，接口响应时间 <= 500ms（AI接口除外）
7. 用户密码加密存储，关键接口鉴权

### 5.3 交付流程

1. **演示阶段**：开发方先将项目部署至服务器，提供访问链接供客户在线体验
2. **验证阶段**：开发方录制本地项目启动视频，证明源码可正常运行
3. **验收阶段**：客户访问在线演示站确认功能无误
4. **交付阶段**：客户确认收货后，开发方交付完整源代码
5. **源码交付**：确认收货后通过约定方式发送前后端完整源码

### 5.4 需求变更规则

- 本文档经双方确认后作为开发及验收的唯一依据
- 超出本文档约定范围的**新增功能**或**功能修改**，视为需求变更
